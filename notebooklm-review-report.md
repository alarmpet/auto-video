# NotebookLM 소스 활용 계획 및 워크플로우 검토 보고서

본 보고서는 `notebooklm-source-plan.md` 계획서와 `auto-video.md` 유튜브 영상 제작 워크플로우를 대조 분석하여, 실제 비디오 제작 파이프라인에서 발생할 수 있는 문제점을 진단하고 실질적인 개선 방안을 제안합니다.

---

## 1. 종합 요약 (Executive Summary)

* **목표의 정합성**: 벤치마킹 채널의 스타일(외형/리듬)에 NotebookLM의 신뢰할 수 있는 소스 지식(성경/심리학)을 결합하여, 할루시네이션(거짓 정보) 없는 60분 이상의 장편 낭독 영상을 제작하는 방향성은 매우 우수합니다.
* **핵심 취약점**: 
  1. NotebookLM 자동화(Playwright)의 빈번한 실패 및 세션 만료에 대한 예외 처리(Fallback)가 워크플로우(`auto-video.md`)에 반영되어 있지 않음.
  2. 60분~90분 분량(1.5만~2.5만 자)의 대본을 한 번에 생성하려고 할 때 발생하는 LLM 출력 토큰 한계(Output Token Limit)와 글자 수 부족 문제.
  3. 90분 영상 제작 시 챕터 개수에 대한 두 문서 간의 지침 불일치.
  4. 벤치마킹 채널의 '자극적/선정적 스타일'과 NotebookLM의 '학술적/신중한 소스 원칙'이 충돌할 때의 우선순위 제어 장치 부족.
  5. 60분 이상의 영상에 필요한 방대한 이미지 프롬프트(30~50개)를 효과적으로 매핑하고 영상 시간(Duration)을 커버할 구조적 대안 부재.

---

## 2. 세부 문제점 및 불일치성 분석 (Detailed Issues)

### ① 분량 및 챕터 수의 상호 불일치 (Chapter Count Discrepancy)
* **현상**: 
  * `auto-video.md` L106: "90분 영상은 14~18개 챕터를 기본으로 한다."
  * `notebooklm-source-plan.md`의 "아담과 하와" 90분 확장 적용안: 기존 13개 챕터 구조를 그대로 유지하되, 각 챕터의 텍스트 길이를 늘려 22,000~25,000자를 달성하라고 가이드함.
* **영향**: 13개 챕터로 25,000자를 채우려면 챕터당 평균 약 1,900자(공백 제외)를 써야 합니다. 이는 낭독 시간으로 챕터당 7~8분에 달하여 시청자가 쉽게 지루해질 수 있으며, 단일 주제를 억지로 늘려 쓰는 느낌을 주기 쉽습니다.

### ② 자동화(Playwright) 신뢰도 한계 및 대비책 부재
* **현상**: NotebookLM은 구글 서비스 특성상 UI 레이아웃이 자주 바뀌며, Playwright 셀렉터가 쉽게 깨집니다. 계획서에서도 "답변 회수 실패 및 세션 만료 현상"을 인지하고 있으나, 정작 메인 지침인 `auto-video.md`에는 이러한 실패 시 사용자가 어떻게 개입해야 하는지(수동 복사-붙여넣기 프로토콜) 전혀 기재되어 있지 않습니다.
* **영향**: 자동화 도구가 에러를 내뿜거나 중간 요약본만 가져왔을 때, Codex가 잘못된 요약본만 가지고 대본을 쓰거나 워크플로우가 중단되는 현상이 발생합니다.

### ③ LLM 출력 토큰 한계로 인한 대본 퀄리티 저하
* **현상**: 최신 LLM도 한 번의 API 호출로 생성할 수 있는 한국어 텍스트는 보통 공백 제외 4,000~6,000자 내외가 한계입니다. 한 번의 프롬프트로 15,000자(60분)나 22,000자(90분)를 한 번에 작성하라고 지시하면, LLM은 앞부분만 자세히 쓰다가 뒷부분을 급격히 축소(생략)하거나 중간에 생성을 멈춰버립니다.
* **영향**: 사용자가 원하는 60분 분량이 나오지 않고, 20~30분 분량으로 타협되거나 챕터 간 분량 불균형이 발생합니다.

### ④ 벤치마킹 스타일 vs 소스 가이드라인의 충돌 (Tone & Policy Override)
* **현상**: 벤치마킹 채널들은 클릭률을 높이기 위해 자극적인 의문문이나 심리 진단적 단정("나르시시스트의 특징", "우울증 환자의 심리")을 남발하는 경향이 있습니다. 반면, NotebookLM 소스 활용 기준은 "단정하지 않고", "나르시시스트처럼 특정하지 않으며", "신학적 해석은 조심스럽게 표현"할 것을 명시합니다.
* **영향**: 스타일 분석 단계(Step 3)에서 벤치마킹 원고를 강하게 모방하다 보면 소스 가이드라인의 윤리적/신학적 원칙을 훼손하는 대본이 생성될 위험이 큽니다.

### ⑤ 비주얼 리소스 매핑의 공백 (Visual Asset Mapping Gap)
* **현상**: 60분 영상에 30~50개 이미지 프롬프트를 생성하면, 이미지 1장당 약 1분~2분 동안 화면에 머물러야 합니다. 3~5초 길이의 비디오 모션 프롬프트를 10개만 만드는 규칙으로는 나머지 50분 이상의 시각적 공백을 채우기 어렵습니다.
* **영향**: 편집 시 영상의 시각적 템포가 너무 느려지거나(정지 화면이 1분 이상 지속), 대본의 흐름과 이미지의 내용이 어긋나는 불일치가 발생합니다.

## 3. 핵심 개선 방안 (Proposed Improvements)

### [개선안 A] 점진적·단계적 대본 집필 워크플로우 도입 (Incremental Chapter Writing)
* 대본을 한 번에 다 쓰는 대신, **"전체 구조 설계 -> 챕터별 점진적 확장 집필 -> 최종 검증 및 조율"** 단계로 나눕니다.
* **1단계**: 10~12개 챕터의 아웃라인과 각 챕터별 핵심 메시지/심리 개념/성경 본문 매핑 테이블 확정.
* **2단계**: 1~4챕터 집필 -> 5~8챕터 집필 -> 9~12챕터 집필식으로 나누어 LLM에 요청하여 출력 토큰 제한을 우회하고 디테일을 극대화합니다.

### [개선안 B] 하이브리드(자동+수동) NotebookLM 프로토콜 명시
* `auto-video.md`에 자동 질의 실패 시의 대비책을 명문화합니다.
* 자동화 도구가 실패하거나 세션 만료 시, 에이전트가 사용자에게 **"NotebookLM에 직접 붙여넣을 완성형 질문 텍스트"**를 화면에 출력해주고, 사용자가 이를 복사하여 NotebookLM 웹 화면에 입력한 뒤 답변을 가져와 붙여넣을 수 있는 수동 포트를 마련합니다.

### [개선안 C] 1회성 마스터 쿼리(Master Query)로 질의 효율화
* 질문을 매번 3~5개씩 쪼개서 하면 수동 흐름에서 사용자 피로도가 극에 달합니다. 
* NotebookLM에 보낼 질문을 **"성경 사건 분석 + 심리학 개념 + 챕터 설계"를 하나로 통합한 고품질 마스터 프롬프트**로 단일화하여 한 번의 질의로 필요한 배경 지식을 모두 수집합니다.

### [개선안 D] 톤앤매너 우선순위(Override Rule) 정의
* "문체와 종결어미, 오프닝 훅의 구조는 벤치마킹 채널을 따르되, **심리 진단적 단정 금지 및 신학적 신중함 등 내용적 원칙은 NotebookLM 가이드라인을 최우선으로 적용한다**"는 명시적 규칙을 `auto-video.md` 상단에 배치합니다.

### [개선안 E] 비주얼 타임라인 배치 및 루핑 가이드 수립
* 이미지 프롬프트를 단순 나열하지 않고, **[챕터 번호 - 대본 구간 - 이미지 프롬프트 - 모션 효과(예: Ken Burns, Slow Zoom)]** 형태로 생성하도록 규격화합니다. 
* 이렇게 하면 30~50개의 리소스가 대본의 어느 타이밍에 들어가는지 명확해지고 편집 효율이 극대화됩니다.

## 4. `auto-video.md` 반영을 위한 프롬프트 업데이트 스펙

`auto-video.md` 문서의 **"3단계: 자막 원고 → 채널 분석 + 문체 추출 + 제목 + 대본"** 섹션에 다음과 같은 수정을 반영할 것을 추천합니다.

```markdown
### [MODIFY] 3단계 워크플로우 세분화 및 하이브리드 소스 입력 경로 추가

1. **벤치마킹 자막 분석 및 문체 추출** (기존 동일)
2. **NotebookLM 기반 지식 추출 (마스터 쿼리 실행)**
   - 에이전트는 NotebookLM 자동 실행을 시도하되, 실패 시 사용자에게 아래 통합 템플릿을 완성하여 제공하고 수동 답변 입력을 대기한다.
   - [통합 템플릿 예시]:
     """
     이 노트북의 소스만을 근거로 삼아, '[영상 주제]'와 관련된 성경 사건/인물의 행동 원인을 현대심리학 개념(예: [심리학 키워드])과 연결하여 설명해 주세요. 
     특히 인물들의 심리적 방어기제나 내면의 갈등(수치심, 두려움 등)을 보여주는 구체적인 근거를 소스에서 찾아 챕터별 이야기 구조로 요약해 주세요.
     """
3. **챕터별 점진적 집필 단계 분할**
   - 60분(15,000자 이상) 장편 대본은 한 번에 쓰지 않고, 3~4개 챕터씩 묶어서 단계적으로 작성한다.
   - 각 단계마다 글자 수를 중간 점검하며 최종 분량을 확보한다.
4. **콘텐츠 정책 필터 적용 (최우선 규칙)**
   - 벤치마킹 채널의 자극적인 어조보다 NotebookLM의 근거 기반 신중한 톤(예: "~로 해석해 볼 수 있습니다", "~의 관점도 존재합니다")을 우위에 둔다.
```
