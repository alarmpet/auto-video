# NotebookLM 소스 활용 계획서

## 목적

`auto-video.md` 워크플로우에 NotebookLM을 연결해, 사용자가 모아 둔 현대심리학·성경 관련 소스를 기반으로 장편 유튜브 낭독 영상을 제작한다.

대상 NotebookLM:

https://notebooklm.google.com/notebook/2b91a71a-2e7d-42f7-a780-ac57d6fc62e3

이 계획의 목표는 단순히 대본을 길게 쓰는 것이 아니라, NotebookLM 안의 소스를 근거로 삼아 다음을 안정적으로 만드는 것이다.

- 벤치마킹 채널식 장편 주제 기획
- 성경 사건과 현대심리학 개념의 연결
- 1시간 이상 낭독 가능한 챕터 구조
- 출처 기반 대본 설계
- 이미지·영상 프롬프트까지 이어지는 제작 파이프라인

## 현재 상태

- `notebooklm` 스킬은 이미 설치되어 있다.
- NotebookLM 인증 상태는 `Authenticated: Yes`로 확인되었다.
- 다만 브라우저 세션이 오래되어 필요 시 재인증이 필요할 수 있다.
- `안티그레비티 / antigravity / anti-gravity` 스킬은 현재 설치된 스킬, 공개 curated 스킬 목록, 검색 가능한 도구 목록에서 발견되지 않았다.
- 따라서 현 단계에서는 NotebookLM 스킬을 중심으로 설계한다.

## 검토 보고서 반영 사항

`notebooklm-review-report.md`를 검토한 결과, 아래 항목은 계획서에 반영할 만큼 타당하다.

- 장편 대본은 한 번에 완성하려 하지 말고, 전체 구조 설계 후 챕터 묶음 단위로 점진 작성한다.
- 90분 이상 영상은 13개 챕터를 단순히 길게 늘리는 방식보다 14~18개 이상의 챕터로 확장하는 편이 안정적이다.
- NotebookLM 자동화 실패에 대비해 수동 복사-붙여넣기 백업 프로토콜을 명확히 둔다.
- 벤치마킹 채널의 클릭 유도 문체보다 성경·심리학 해석의 정확성과 신중함을 우선한다.
- 이미지 프롬프트는 단순 나열이 아니라 챕터, 대본 구간, 예상 시점, 이미지, 모션 효과를 함께 묶는 타임라인 형태로 관리한다.

아래 항목은 부분 반영한다.

- “마스터 쿼리 1회로 모든 질문을 단일화”는 초반 개요 추출에는 유용하지만, NotebookLM 응답 누락이나 과도한 요약 위험이 있으므로 후속 세부 질문과 함께 사용한다.
- “LLM 출력 한계”는 모델·환경에 따라 달라지는 절대 수치가 아니므로, 긴 대본에서 축약·중단·챕터 불균형이 생길 수 있는 운영 리스크로 취급한다.

## 핵심 운영 원칙

NotebookLM은 “대본을 대신 쓰는 도구”가 아니라 “소스 기반 리서치와 검증 도구”로 사용한다.

제작 흐름은 다음과 같이 나눈다.

1. `Codex`: 채널 분석, 제목 패턴 분석, 대본 구조 설계, 최종 문체 적용
2. `NotebookLM`: 현대심리학·성경 자료에서 근거, 개념, 연결점, 해석 포인트 추출
3. `auto-video.md`: 단계별 사용자 입력과 산출물 형식 관리

즉, NotebookLM에서 나온 내용을 그대로 복붙하지 않고, 소스 기반 재료로만 사용한다.

## auto-video.md에 추가할 활용 흐름

### 1단계: 채널 스크린샷 분석

기존 흐름대로 진행한다.

- 인기 영상 제목·조회수·업로드 시기 정리
- 전문 분야·대상 시청자 분석
- 제목 패턴 분석
- 영상 아이디어 10개 제안

이 단계에서는 NotebookLM을 사용하지 않는다.

### 2단계: 주제 선택

사용자가 주제를 고르면, 기존처럼 벤치마킹 채널 자막 원고를 요청한다.

단, 주제가 성경·심리학·철학·역사처럼 근거가 중요한 분야라면 다음 문장을 추가한다.

> 이 주제와 관련해 NotebookLM에 모아 둔 소스도 함께 활용하겠습니다. 대본을 쓰기 전에 NotebookLM에서 성경 본문, 심리학 개념, 해석 포인트를 먼저 추출해 근거 기반 구조를 만들겠습니다.

### 3단계: 자막 원고 + NotebookLM 소스 기반 대본 작성

기존 3단계에 NotebookLM 리서치 단계를 추가한다.

순서는 다음과 같다.

1. 벤치마킹 자막 원고 분석
2. 채널 문체·리듬·감정 곡선 추출
3. NotebookLM에 마스터 쿼리 실행
4. 부족한 부분은 NotebookLM 후속 질문으로 보강
5. 성경 사건·심리학 개념·해석 포인트 정리
6. 장편 챕터 구조 설계
7. 제목 후보 5개 생성
8. 60분 이상 목표 글자 수 설정
9. 챕터 묶음 단위로 점진적 대본 작성
10. 전체 대본 연결부, 중복, 분량 균형 조율
11. 최종 글자 수와 예상 영상 길이 표시

### 점진적 장편 대본 작성 방식

60분 이상 장편 대본은 한 번에 전부 쓰지 않는다. 긴 출력에서는 뒤쪽 챕터가 급격히 축약되거나 중간에 생략될 수 있으므로 아래 방식으로 나눠 작성한다.

1. 전체 챕터 구조와 각 챕터의 핵심 메시지를 먼저 확정한다.
2. 챕터별 목표 글자 수를 배정한다.
3. 60분 영상은 3~4개 챕터씩 묶어 작성한다.
4. 90분 영상은 3개 챕터씩 묶어 작성하거나, 챕터별 단독 작성으로 진행한다.
5. 각 묶음 작성 후 누적 글자 수를 점검한다.
6. 마지막에 전체 흐름, 반복 표현, 감정 곡선, 다음 챕터 연결부를 조율한다.

권장 챕터 수는 다음과 같이 조정한다.

| 목표 영상 길이 | 권장 챕터 수 | 운영 기준 |
|---|---:|---|
| 40분 | 8~10개 | 챕터당 4~5분 |
| 60분 | 10~13개 | 챕터당 4~6분 |
| 90분 | 14~18개 | 챕터당 5~6분 |
| 120분 | 18~22개 | 챕터당 5~7분 |

13개 이하 챕터로 90분 이상을 만들 경우, 한 챕터가 지나치게 길어져 반복감이 생길 수 있으므로 새 장면, 새 질문, 새 심리 개념을 추가해 챕터를 늘리는 것을 우선한다.

## NotebookLM 질문 템플릿

아래 질문들은 장편 낭독 대본을 만들 때 반복해서 사용한다.

### 마스터 쿼리

```text
이 노트북의 소스만 근거로 삼아, “[영상 주제]”를 60분 이상 유튜브 장편 낭독 영상으로 만들기 위한 리서치 노트를 작성해 주세요.

반드시 아래 항목으로 나눠 주세요.

1. 관련 성경 본문과 사건 흐름
2. 연결 가능한 현대심리학 개념
3. 인물의 행동 원인을 해석할 수 있는 심리적 동기
4. 수치심, 죄책감, 불안, 욕망, 책임 전가, 자기기만, 관계 갈등과 연결되는 지점
5. 10~13개 챕터 구조
6. 각 챕터별 핵심 질문, 핵심 장면, 핵심 메시지
7. 신학적으로 조심해야 할 해석
8. 심리학적으로 단정하면 안 되는 표현
9. 대본에 넣기 좋은 차분한 낭독형 문장 소재
```

마스터 쿼리 응답이 너무 추상적이거나 짧으면 아래 세부 질문으로 나눠 다시 묻는다.

### 주제 개요 추출

```text
이 노트북의 소스만 근거로 삼아, “[영상 주제]”와 관련된 핵심 성경 본문, 현대심리학 개념, 주요 해석 포인트를 정리해 주세요. 유튜브 장편 낭독 대본에 사용할 수 있도록 챕터별로 나눠 주세요.
```

### 성경 사건 분석

```text
“[성경 사건/인물]”을 현대심리학 관점에서 해석할 때 사용할 수 있는 근거를 이 노트북의 소스에서 찾아 정리해 주세요. 욕망, 수치심, 죄책감, 책임 전가, 불안, 관계 갈등 같은 개념과 연결해 주세요.
```

### 심리학 개념 추출

```text
이 노트북의 소스에서 “[심리학 개념]”과 관련된 설명을 찾아 주세요. 성경 이야기와 연결할 수 있는 포인트, 조심해야 할 오해, 대본에 넣기 좋은 표현을 구분해 주세요.
```

### 장편 챕터 설계

```text
“[영상 주제]”를 60분 이상 유튜브 낭독 영상으로 만들려고 합니다. 이 노트북의 소스에 근거해 10~13개 챕터 구조를 제안해 주세요. 각 챕터마다 성경 장면, 심리학 개념, 핵심 질문, 대본에 넣을 메시지를 포함해 주세요.
```

### 90분 이상 챕터 확장

```text
“[영상 주제]”를 90분 이상 장편 낭독 영상으로 확장하려고 합니다. 기존 10~13개 챕터를 억지로 길게 늘리지 말고, 이 노트북의 소스에 근거해 14~18개 챕터로 자연스럽게 확장해 주세요. 각 추가 챕터는 새 장면, 새 질문, 새 심리 개념을 가져야 합니다.
```

### 근거 검증

```text
다음 해석이 이 노트북의 소스와 일치하는지 검토해 주세요. 과장된 부분, 근거가 약한 부분, 신학적으로 조심해야 할 부분, 심리학적으로 부정확한 부분을 나눠서 지적해 주세요: [해석 내용]
```

### 대본 보강

```text
다음 대본 초안을 NotebookLM 소스 기준으로 보강하려고 합니다. 빠진 성경적 근거, 더 적절한 심리학 개념, 장편 낭독에 맞는 추가 챕터 아이디어를 제안해 주세요: [대본 초안]
```

### 제목 검증

```text
다음 제목 후보들이 이 노트북의 소스와 주제에 잘 맞는지 평가해 주세요. 클릭 유도는 강하지만 왜곡되거나 자극적인 제목은 제외하고, 성경과 심리학의 균형이 좋은 제목을 추천해 주세요: [제목 후보]
```

### 이미지 장면 추출

```text
“[영상 주제]” 대본을 이미지 프롬프트로 만들기 위해, 이 노트북의 소스에 근거한 핵심 장면 30~50개를 제안해 주세요. 각 장면은 성경적 배경, 인물의 감정, 상징물, 분위기를 포함해 주세요.
```

## “아담과 하와 / 선악과” 주제 적용안

현재 작업 중인 주제:

> 잠들기 전 듣는 성경 속 인간 심리, 아담과 하와가 선악과를 따먹은 진짜 이유

이 주제는 NotebookLM을 활용해 아래 구조로 확장한다. 아래 13개 챕터는 60분용 기본 구조이며, 90분 이상으로 확장할 때는 뒤의 확장 챕터를 추가한다.

1. 프롤로그: 에덴의 밤
2. 에덴동산은 정말 완전한 낙원이었을까
3. 인간에게 자유와 금지는 왜 함께 주어졌을까
4. 뱀은 왜 명령하지 않고 질문했을까
5. 하와는 왜 선악과를 다르게 보기 시작했을까
6. 아담은 왜 침묵했을까
7. 눈이 밝아진 뒤 찾아온 것은 지혜였을까, 수치심이었을까
8. 무화과 잎은 인간의 첫 번째 방어기제였을까
9. 하나님은 왜 “네가 어디 있느냐”라고 물었을까
10. 아담은 왜 하와를 탓했고, 하와는 왜 뱀을 탓했을까
11. 에덴 밖의 삶은 형벌이었을까, 성장의 시작이었을까
12. 현대인의 작은 선악과: 비교, 인정 욕구, 자기기만, 불안
13. 에필로그: 우리는 어디에 숨어 있는가

60분 영상 목표라면 공백 제외 15,000~17,000자를 목표로 한다.

90분 영상 목표라면 단순히 각 챕터를 길게 늘리지 않고, 아래와 같은 확장 챕터를 추가해 14~18개 챕터로 늘린다.

14. 금지 효과: 하지 말라는 말은 왜 욕망을 키우는가
15. 관계 안의 침묵: 아담은 왜 멈추지 않았는가
16. 무화과 잎의 심리학: 인간은 무엇으로 자신을 가리는가
17. 에덴 이후의 관계: 사랑은 왜 지배와 두려움으로 변하는가
18. 회복의 시작: 숨은 자리에서 나오는 마음

90분 영상은 공백 제외 22,000~25,000자를 목표로 한다.

## 소스 활용 기준

NotebookLM에서 얻은 내용은 다음 기준으로 분류한다.

| 분류 | 사용 방식 |
|---|---|
| 성경 본문 | 사건 순서와 핵심 장면의 기준으로 사용 |
| 심리학 개념 | 장면을 해석하는 렌즈로 사용 |
| 신학적 해석 | 단정하지 않고 “이렇게 읽을 수도 있습니다” 방식으로 사용 |
| 현대 사례 | 수면 낭독 분위기를 해치지 않는 선에서 짧게 사용 |
| 논쟁적 내용 | 출처가 충분할 때만 조심스럽게 사용 |

## 톤앤매너 우선순위

벤치마킹 채널의 문체와 NotebookLM 소스 원칙이 충돌할 때는 아래 우선순위를 따른다.

1. 성경 본문과 출처 기반 정확성
2. 신학적·심리학적 신중함
3. 수면용 낭독에 맞는 차분한 정서
4. 벤치마킹 채널의 문장 리듬과 오프닝 구조
5. 클릭률을 위한 제목·썸네일식 자극

따라서 벤치마킹 채널이 강한 단정형 표현을 쓰더라도, 성경·심리학 주제에서는 다음 표현을 우선한다.

- “~라고 단정할 수 있습니다”보다 “~로 해석해 볼 수 있습니다”
- “이 인물은 나르시시스트입니다”보다 “인정 욕구와 자기중심성이 강하게 드러나는 장면으로 읽을 수 있습니다”
- “성경이 현대 심리학을 증명합니다”보다 “현대 심리학의 언어로 다시 비춰볼 수 있습니다”
- “이것이 정답입니다”보다 “이 관점은 한 가지 중요한 단서를 제공합니다”

## 주의점

- 성경 본문을 현대심리학으로 과도하게 환원하지 않는다.
- 심리학 개념을 진단처럼 사용하지 않는다.
- 특정 인물을 “나르시시스트”, “트라우마 환자”처럼 단정하지 않는다.
- 신학적으로 민감한 해석은 절대적 결론이 아니라 가능한 해석으로 표현한다.
- NotebookLM 응답이 불완전하면 같은 질문을 더 구체적으로 다시 묻는다.
- NotebookLM 자동화가 답변을 제대로 회수하지 못하면, 사용자가 NotebookLM 화면에서 답변을 복사해 붙여넣는 수동 백업 흐름을 사용한다.

## 자동화 리스크와 대응

현재 NotebookLM 자동 질의는 접속과 입력은 가능하지만, 답변 회수 단계에서 최종 답변 대신 중간 요약 문장만 반환되는 현상이 있었다.

대응 방법:

1. 질문을 더 짧고 구체적으로 나눠 다시 묻는다.
2. `--show-browser` 옵션으로 실제 브라우저 화면을 확인한다.
3. 인증 세션이 오래되었으므로 문제가 반복되면 재인증한다.
4. 자동 회수가 계속 실패하면 NotebookLM 답변을 사용자가 복사해 붙여넣고, Codex가 그 내용을 분석한다.

### 하이브리드 NotebookLM 프로토콜

NotebookLM 자동화는 항상 먼저 시도하되, 아래 상황에서는 수동 백업 흐름으로 전환한다.

- 로그인 세션 만료로 NotebookLM 접근이 막힐 때
- 자동화가 답변 본문 대신 중간 요약이나 로딩 문구만 가져올 때
- NotebookLM UI 변경으로 입력창이나 답변 영역을 찾지 못할 때
- 답변이 지나치게 짧아 소스 기반 구조를 만들기 어려울 때

수동 백업 흐름:

1. Codex가 NotebookLM에 붙여넣을 완성형 질문을 제공한다.
2. 사용자가 NotebookLM 웹 화면에 질문을 직접 붙여넣는다.
3. 사용자가 NotebookLM 답변 전체를 Codex에 붙여넣는다.
4. Codex가 답변을 요약하지 않고, 근거·개념·챕터·주의점으로 재분류한다.
5. 필요한 경우 Codex가 후속 질문을 다시 작성해 제공한다.

이 방식은 자동화를 포기하는 것이 아니라, 구글 UI 변경과 세션 만료로 인한 제작 중단을 막기 위한 안전장치다.

## 비주얼 타임라인 운영

60분 이상 장편 영상에서는 이미지 프롬프트를 단순 나열하지 않고 타임라인으로 관리한다.

기본 형식:

```text
[챕터 번호] [예상 시점] [대본 구간] 이미지 프롬프트 / 카메라 앵글 / 조명 / 분위기 / 동작 / 권장 모션
```

운영 기준:

- 각 챕터마다 핵심 이미지 3~5개를 먼저 만든다.
- 전체 영상용 이미지는 60분 기준 40~70개를 1차 목표로 한다.
- 세밀한 컷 분할이 필요한 경우에만 3~5초 단위 프롬프트로 확장한다.
- 정지 이미지가 오래 머무르는 구간은 slow zoom, slow pan, parallax, light flicker 같은 반복 가능한 모션을 지정한다.
- 앞 10개 영상 프롬프트만 별도로 만드는 기존 규칙은 유지하되, 장편 영상에서는 각 챕터 대표 장면에도 권장 모션을 함께 적는다.
- 대본 전체를 빠뜨리지 않기 위해 이미지마다 연결되는 대본 구간을 반드시 표시한다.
