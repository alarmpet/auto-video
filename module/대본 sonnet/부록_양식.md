## 1. 인트로 자가검증 표 (4-A 단계 의무)

작성 후 아래 표를 자체 채움. **하나라도 ❌면 자동 폐기·재작성. 통과 시 표 생략하고 다음 단계.**

| 점검 항목 | 결과 |
|---|---|
| 스포일러 Q1 (결말 짐작 가능?) | ✅ NO / ❌ YES |
| 스포일러 Q2 (정체·신분 노출?) | ✅ NO / ❌ YES |
| 스포일러 Q3 (범인 감?) | ✅ NO / ❌ YES |
| 스포일러 Q4 (해결 노출?) | ✅ NO / ❌ YES |
| 스포일러 Q5 (제목 행동의 동기·결과·정체·반전 노출?) | ✅ NO / ❌ YES |
| 닫힌 질문 회피 | ✅ / ❌ |
| 어미 인접 반복 (~을까요/~지요 3+) | ✅ / ❌ |
| 글자수 200~350 | ✅ / ❌ |
| 6문장 포맷 | ✅ / ❌ |
| 1문장 임팩트 (큰따옴표·날것 감정) | ✅ / ❌ |

## 5. 반전·발단 자가검증 5문항 (4-C 단계 의무, 각 반전·발단마다)

각 반전마다 5문항 답을 명시. **하나라도 NO면 그 반전 폐기·재설계.**
**Q5는 발단·갈등 구조에도 동일 적용 (사건 인과의 논리 점검).**

| 반전 # | Q1: 등장인물 X가 모를 이유 있는가? | Q2: 본문에서 자연스럽게 도출 가능한가? | Q3: 다른 반전과 정보 겹치지 않는가? | Q4: 인트로·제목에 노출 안 됐는가? | Q5: 인물 행동·상황 인과가 논리적으로 성립하는가? |
|---|---|---|---|---|---|
| 1 | YES/NO | YES/NO | YES/NO | YES/NO | YES/NO |
| 2 | | | | | |
| 3 | | | | | |
| 4 | | | | | |
| 5 | | | | | |
| 6 | | | | | |

### Q5 점검 예시 (꼭 확인할 패턴)

- "맹인이라 반박 불가" — 맹인은 목격 증언이 안 되니 오히려 고발 측이 다른 근거 필요. 논리 역방향 ❌
- "정혼녀면 가문 딸인 거 모름" — 정혼이면 당연히 안다. 논리 모순 ❌
- "스스로 들어온 자를 처음부터 모름" — 스스로 들어왔으면 받아준 측이 안다. 논리 모순 ❌
- "첩이 첩인 줄 모름" — 첩 관계는 양쪽 다 안다. 논리 모순 ❌

### 발단 자가검증 (별도 적용)

발단 구도 작성 후 Q5 단독 적용:
- 악역의 동기가 그 행동을 선택한 이유로 충분한가?
- 주인공의 약점·강점이 사건 발생 이유와 일관성 있는가?
- 시간·공간 제약 안에서 그 행동이 가능한가?

### 4가지 카테고리 분산 검증

```
신분 반전: __개
능력 반전: __개
관계 반전: __개
초자연·운명 반전: __개

→ 한 카테고리 4개 이상 = ❌ (분산 실패, 재설계)
```

## 8. 챕터 검증 체크리스트 (6단계 의무, 매 챕터 후) — ★ Opus 모드 + 티어별 ★

scripts.md의 `check_chapter_lite(text, target, tier, facts_dict)` 함수 자동 실행.
**치명적 5개 + 티어별 글자수 검증.**
문체·TTS 검증은 합본 후 `check_final`에서 일괄.

**Opus 모드 챕터 단위 검증:**

| 항목 | normal (1h/1.5h) | strict (2h) | 결과 |
|---|---|---|---|
| 한자 잔존 | 동일 | 동일 | ✅ / ❌ |
| 메타 정보 | 동일 | 동일 | ✅ / ❌ |
| 따옴표 짝 | 동일 | 동일 | ✅ / ❌ |
| 인접 동일 대사 | 동일 | 동일 | ✅ / ❌ |
| 글자수 | ±25%, 바닥 3,500 | ★ ±15%, 바닥 4,410 | ✅ ____자 / ❌ |
| 나이 표기 (선택) | 동일 | 동일 | ✅ / ❌ |

**★ strict 추가 — 누적 페이스 체크:**
| 항목 | 결과 |
|---|---|
| 누적 글자수 | ____자 |
| 예상 페이스 | ____자 |
| 차이 | +/-___% |
| 페이스 경고 (-5% 이하) | ✅ / ⚠️ |

❌ 1건이라도 → 부분 수정 1회만.
**부분 수정 1회 = 절대 한도. 무한 루프 금지.** 안 되면 사용자 보고.

## 11. 사용자 단어 사전

| 사용자 입력 | 의미 |
|---|---|
| "다시" | 전면 재시도 |
| "별로" | 그 항목만 거부, 다른 옵션 |
| "이상해" | 결함 지적, 곧 구체 지적 따라옴 |
| "별론데" | 부드러운 거부 + 이유 설명 |
| "어색해" | 표현 문제, 단어 교체 |
| "근데" + 지적 | 발견된 모순 짚기 |
| "응" / "굿" / "OK" | 즉시 다음 단계 |
| "1" / "A" / "B" | 옵션 선택 |
| "더 자극적으로" / "임팩트 잇게" | 톤 강화 |
| "계속" | 작업 디렉토리에 progress.md 있으면 끊김 재개 / 없으면 디폴트 진행 |
